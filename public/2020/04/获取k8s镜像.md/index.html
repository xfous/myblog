获取k8s镜像.md

2020-04-05 发布在 k8s

由于国内糟糕的网络环境，不能正常获取镜像。可以用下面的方法解决

目录

  • 使用加速镜像源pull到本地
    • Azure 中国镜像源
    • 中科大镜像源
  • 使用mirrorgooglecontainers用户同步的镜像
  • 从阿里云镜像仓库拉取
  • 自动化工具

使用加速镜像源pull到本地

  • Azure 中国：http://mirror.azure.cn/
  • Azure 中国 Github：https://github.com/Azure/container-service-for-azure-china
  • 配置说明：http://mirror.azure.cn/help/gcr-proxy-cache.html
  • 中科大：http://mirrors.ustc.edu.cn/
  • 中科大 Github：https://github.com/ustclug/mirrorrequest
  • 配置说明：https://github.com/ustclug/mirrorrequest/issues/18

Azure 中国镜像源

  • docker.io镜像加速

    docker pull aaa:bbb
docker pull dockerhub.azk8s.cn/library/<imagename>:<version>
docker pull dockerhub.azk8s.cn/library/nginx:1.13

  • gcr.io镜像加速

    docker pull gcr.azk8s.cn/google_containers/<imagename>:<version>
docker pull k8s.gcr.io/pause-amd64:3.0
docker pull gcr.azk8s.cn/google_containers/pause-amd64:3.0

  • quay.io镜像加速

    docker pull quay.azk8s.cn/<imagename>:<version>
docker pull quay.io/coreos/etcd:v.3.4.1
docker pull quay.azk8s.cn/coreos/etcd:v.3.4.1

中科大镜像源

  • docker.io镜像加速

    docker pull dockerhub.azk8s.cn/library/nginx:1.13

  • gcr.io镜像加速

    docker pull gcr.mirrors.ustc.edu.cn/pause-amd64:3.0

  • quay.io镜像加速

    docker pull quay.mirrors.ustc.edu.cn/coreos/etcd:v.3.4.1

使用mirrorgooglecontainers用户同步的镜像

mirrorgooglecontainers用户在docker hub同步了所有k8s最新的镜像,同步后改tag
https://hub.docker.com/u/mirrorgooglecontainers

从阿里云镜像仓库拉取

docker pull registry.cn-hangzhou.aliyuncs.com/google_containers/kube-apiserver-amd64:v1.13.0

自动化工具

已经有大神写了工具可以直接使用

  • azk8spull：https://github.com/xuxinkun/littleTools
  • getgcr：https://git.io/getgcr
  • docker-wrapper：https://github.com/silenceshell/docker_wrapper
